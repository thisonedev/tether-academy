Now that we can do an in-memory search, we're going to add persistence.

The tiny search engine from chapter 3 lived entirely in memory, and the corpus was rebuilt every run. RAG (Retrieval-Augmented Generation) gives us a workspace: a folder on disk where vectors are stored between runs, indexed by name, and ready to query. The `rag*` family covers the full lifecycle: ingest, search, chunk, reindex, list, save pre-computed vectors, delete individual entries, drop a whole workspace. When the built-in store isn't enough, the same `embed()` call feeds an external vector DB.
