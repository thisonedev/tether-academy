The previous lesson paired ESRGAN with a diffusion model. You can also load ESRGAN on its own and run it against any PNG or JPEG, generated or not.

The path is the same `loadModel` call with two changes: the model is the upscaler itself, and `modelConfig.mode` is `"upscale"` instead of the default `"diffusion"`. From there, you call `upscale()` instead of `diffusion()`. The model never sees a prompt, and the upscaler never sees a diffusion pass.

Loading the upscaler with `mode: "upscale"` is how the same ESRGAN engine knows to upscale instead of generating from noise. `modelType: "diffusion"` is still the right type because the model is loaded the same way; only `modelConfig.mode` changes. `tile_size` is a memory knob you set once at load time. Here's how you would put it into code:

```ts
const modelId = await loadModel({
  modelSrc: REALESRGAN_X4PLUS_ANIME_6B,
  modelType: "diffusion",
  modelConfig: {
    mode: "upscale",
    upscaler: { tile_size: 128 },
  },
});
```

The engine reads bytes, not filenames, so the source can be a hand-drawn sketch, a screenshot, or a generated image:

```ts
const pngBytes = new Uint8Array(fs.readFileSync("./examples/diffusion/input/sketch.png"));
```

One PNG at `source * scale^repeats` dimensions; intermediate passes are internal:

```ts
const result = upscale({ modelId, image: pngBytes, repeats: 2 });
const outputs = await result.outputs;
const first = outputs[0];
if (!first) throw new Error("No image returned from upscale");
fs.writeFileSync("sketch_x16.png", first);
console.log(`Generated ${outputs.length} image`);
```

The ESRGAN model handles the entire job. Diffusion never runs. The `repeats` count is the only knob; `tile_size` is a memory knob you set once at load time.

> Note: `upscale()` returns one PNG even when `repeats > 1`. The intermediate passes are internal. If you need each pass as a separate image, use the diffusion-paired path from the previous lesson.

## Put it to the test

1. Call `loadModel` with `modelType: "diffusion"` and `modelConfig: { mode: "upscale", upscaler: { tile_size: 128 } }`.
2. Read the source PNG into a `Uint8Array` with `fs.readFileSync`.
3. Call `upscale({ modelId, image, repeats: 2 })`, await `result.outputs`, write `outputs[0]` to a PNG, and console.log the count.
