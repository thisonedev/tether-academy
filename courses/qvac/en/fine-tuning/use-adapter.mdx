Now that we know how to train an adapter, we're going to use it.

The trained adapter is just a small `.gguf` file. The base model stays the same. Stacking the adapter happens at `loadModel()` by passing `modelConfig.lora` with the file path.

Loading the base model with `modelConfig.lora` pointing at the adapter file is the inference setup. The base model stays the same; we just stack the trained LoRA on top. The adapter is a small `.gguf` file produced by `finetune()`:

```ts
const modelId = await loadModel({
  modelSrc: QWEN3_600M_INST_Q4,
  modelConfig: {
    lora: "./examples/finetune/results/adapter.gguf",
  },
});
console.log("Loaded with adapter");
```

Building the history with one user message is the same shape as a non-adapted call. The adapter doesn't change the prompt format:

```ts
const history = [{ role: "user", content: "What's a peanut butter sandwich in one sentence?" }];
```

Calling `completion()` with `stream: true` and streaming the tokens to stdout is the visible part. The trained behavior shows up token by token. Every call against this `modelId` reflects the trained behavior; there's nothing to set on `completion()` itself. Putting it into code:

```ts
const result = completion({
  modelId,
  history,
  stream: true,
});

for await (const token of result.tokenStream) {
  process.stdout.write(token);
}
process.stdout.write("\n");
```

> Note: the lora field can be re-set on a new `loadModel` call to switch adapters. Reload with a different path.

## Put it to the test

1. Call `loadModel` with `modelConfig.lora` pointing at a `.gguf` adapter path, and console.log `"Loaded with adapter"`.
2. Build a `history` array with one user message.
3. Call `completion({ modelId, history, stream: true })` and iterate `result.tokenStream` to stdout.
